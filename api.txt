				Описание API сервера чата

			Основные принципы работы с API

Для обмена служебными сообщениями между клиентом и сервером используется формат XML. Кодировка сообщений - UTF-8.
При отправке сообщений серверу, служебное сообщение должно быть обрамлено тэгом <chat> (см. пример запроса к api).
Сообщение не должно содержать символов перевода строки (он должен стоять только в конце сообщения).

			Допустимые действия (action) при запросе API
Действие 		 Описание        		Параметры запроса  		Тэги ответа
reg     	регистрация       			username,password 		status
login   	вход пользователя       		username,password   		session_id/status
logout  	выход пользователя      		username,session_id  		status
get_users	Получить список пользователей 		username,session_id  		user (по количеству пользователей) с тэгом status
msg-to  	сообщение для пользователя     		to (username),session_id	status
			Допустимые ответы сервера
Тэг			Описание					Параметры
action type="msg-from"  сообщение для пользователя 		 from (username),message
action type="broadcast" сообщение о статусе пользователя	 username,status			
session_id		id сессии при успешной авторизации	 id сессии в тэге
status 			Статус запроса				 Код статуса и его тип (в аттрибуте type тэга)	
	
				Коды статуса
  0 - Действие успешно выполнено
  1 - При выполнении запроса не указан один из параметров
  2 - Пользователь с таким логином существует
  3 - Неверные имя пользователя и пароль
  4 - Ошибка проверки подлинности (попытка выполнения команды от имени другого пользователя)
  5 - Пользователь заблокирован
  6 - Пользователь не существует

		Коды статуса для широковещательных сообщений и тэга status
  0 - Пользователь оффлайн
  1 - Пользователь онлайн

				Пример запроса API 
Переводы строки добавлены для улучшения читаемости запроса
Запрос регистрации нового пользователя:
<?xml version="1.0" encoding="UTF-8" ?>
<chat>
<action type="reg">
<param name="username">123</param>
<param name="password">1234</param>
</chat>
				Пример ответа сервера
Переводы строк добавлены для улучшения читаемости запроса
Сообщение статуса
<?xml version="1.0" encoding="UTF-8"?>
<status type="OK">0</status>
